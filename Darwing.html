Conceptos Básicos De Programación

  • Algoritmos
    • Definición de Algoritmo: Un conjunto de instrucciones paso a paso para resolver un problema o realizar una tarea.
    • Características de los Algoritmos:
      • Finitud: Debe terminar después de un número finito de pasos.
      • Claridad: Debe ser claro y sin ambigüedades.
      • Eficiencia: Debe ser eficiente en términos de tiempo y recursos.
  • Arrays
    • Declaración y Uso: Definir y utilizar arrays en la programación.
    • Acceso a Elementos: Acceder a elementos en una array mediante índices.
    • Manipulación de Arrays: Modificar y gestionar datos dentro de arrays.
  • Bucles (Loops)
    • For: Iterar un bloque de código un número determinado de veces.
    • While: Repetir un bloque de código mientras una condición sea verdadera.
    • Do-While: Ejecutar un bloque de código al menos una vez y luego repetir mientras una condición sea verdadera.
  • Clases y Objetos (Programación Orientada a Objetos)
    • Definición de Clase: Plantilla para crear objetos con propiedades y métodos.
    • Instanciación de Objetos: Crear instancias de una clase.
    • Métodos y Atributos: Funciones y variables dentro de una clase.
  • Condicionales
    • If: Ejecutar un bloque de código si una condición es verdadera.
    • Else: Ejecutar un bloque de código si la condición del if es falsa.
    • Else If: Probar una condición adicional si la anterior es falsa.
    • Switch: Seleccionar entre múltiples bloques de código a ejecutar.
  • Depuración y Pruebas
    • Errores de Sintaxis: Violaciones de las reglas del lenguaje de programación.
    • Errores Lógicos: Fallos en la lógica del código que producen resultados incorrectos.
    • Pruebas Unitarias: Verificar que cada unidad individual de código funcione correctamente.
  • Estructuras de Datos
    • Listas: Colección ordenada de elementos que pueden ser de diferentes tipos.
    • Pilas: Estructura de datos LIFO (Last In, First Out).
    • Colas: Estructura de datos FIFO (First In, First Out).
    • Grafos: Conjunto de nodos conectados por aristas.
    • Árboles: Estructura jerárquica con un nodo raíz y nodos hijos.
  • Funciones
    • Declaración de Funciones: Definir una función para realizar una tarea específica.
    • Parámetros y Argumentos: Variables y valores pasados a una función.
    • Retorno de Valores: Devolver un valor desde una función al código que la llamó.
  • Herencia (Programación Orientada a Objetos)
    • Clases Base y Derivadas: Definir clases que heredan propiedades y métodos de otra clase.
    • Sobrescritura de Métodos: Redefinir un método en una clase derivada.
  • Interfaces (Programación Orientada a Objetos)
    • Definición y Uso: Definir un contrato que las clases deben implementar.
    • Implementación de Interfaces: Crear clases que cumplan con los métodos definidos en una interfaz.
  • Métodos (Programación Orientada a Objetos)
    • Métodos de Instancia: Métodos que operan en instancias de una clase.
    • Métodos Estáticos: Métodos que pertenecen a la clase en sí y no a instancias específicas.
    • Sobrecarga de Métodos: Definir múltiples métodos con el mismo nombre pero diferentes parámetros.
  • Polimorfismo (Programación Orientada a Objetos)
    • Polimorfismo en Tiempo de Compilación: Sobrecarga de métodos.
    • Polimorfismo en Tiempo de Ejecución: Sobrescritura de métodos.
  • Recursión
    • Definición y Uso: Función que se llama a sí misma para resolver un problema.
    • Ejemplos de Recursión: Problemas clásicos como el factorial o la secuencia de Fibonacci.
    • Ventajas y Desventajas: Simplifica el código para algunos problemas, pero puede ser menos eficiente.
  • Tipos de Datos
    • Enteros: Números sin parte decimal.
    • Flotantes: Números con parte decimal.
    • Caracteres: Símbolos individuales.
    • Booleanos: Valores verdaderos o falsos.
    • Cadenas de Texto: Secuencias de caracteres.
  • Variables
    • Declaración de Variables: Definir variables para almacenar datos.
    • Tipos de Variables: Diferentes tipos de datos que las variables pueden almacenar.
    • Alcance de Variables: El contexto en el que una variable está disponible para su uso.
  • Entrada y Salida
    • Lectura de Datos: Recibir información del usuario o de dispositivos externos.
    • Escritura de Datos: Enviar información al usuario o a dispositivos externos.

Variables

  • Concepto
    • Una variable es un contenedor o un espacio de almacenamiento en el que puedes guardar y manipular datos en un programa de computadora. En términos más simples, podrías pensar en una variable como una etiqueta o un nombre asociado a un valor que puede cambiar durante la ejecución de un programa. Cada variable tiene un nombre único que se utiliza para referirse a ella dentro del programa. Este nombre es utilizado para acceder al valor almacenado en la variable y para realizar operaciones con ese valor.
    • Ejemplo:

      • Edad = 25

      Nombre = "Juan"
